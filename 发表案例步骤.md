## 第一步：创建远程仓库

### 方案A：使用GitHub（推荐）

1. **访问GitHub**
   - 打开 https://github.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角 "+" → "New repository"
   - 填写仓库信息：
     - **Repository name**: `CampusSpace`
     - **Description**: `校园场地智能预约管理系统 - Django 3.2 LTS`
     - **Public**（必须公开，才能分享到社区）
     - **不要勾选** "Add a README file"（我们已经有了）
     - **不要勾选** "Add .gitignore"（我们已经有了）
   - 点击 "Create repository"

3. **记录仓库地址**
   - 创建后会显示仓库地址，例如：
     - HTTPS: `https://github.com/你的用户名/CampusSpace.git`
     - SSH: `git@github.com:你的用户名/CampusSpace.git`

### 方案B：使用Gitee（国内访问更快）

1. **访问Gitee**
   - 打开 https://gitee.com
   - 登录你的账号

2. **创建新仓库**
   - 点击右上角 "+" → "新建仓库"
   - 填写仓库信息：
     - **仓库名称**: `CampusSpace`
     - **仓库介绍**: `校园场地智能预约管理系统 - Django 3.2 LTS`
     - **公开**（必须公开）
     - **不要勾选** "使用Readme文件初始化这个仓库"
   - 点击 "创建"

3. **记录仓库地址**
   - 例如：`https://gitee.com/你的用户名/CampusSpace.git`

## 📝 案例文章模板

```markdown
# CampusSpace校园场地智能预约管理系统

## 项目介绍

CampusSpace是一个基于Django 3.2 LTS开发的校园场地智能预约管理系统，提供场地预约、审批流程、智能推荐、冲突检测等功能。

## 技术栈

- 后端框架：Django 3.2 LTS
- 前端框架：Bootstrap 5
- 数据库：SQLite（开发）/ PostgreSQL（生产）
- 缓存：Redis
- API文档：drf-yasg
- 对象存储：华为云OBS

## 核心功能

### 1. 场地管理
- 楼宇、场地、设备的增删改查
- 场地状态管理（可用、维护中、已停用）
- 设备关联与管理

### 2. 预约管理
- 创建、查询、修改、取消预约
- 预约状态流转（待审批→已通过→已完成）
- 预约历史记录

### 3. 审批流程
- 多级审批支持
- 批量审批功能
- 审批意见记录
- 审批历史查询

### 4. 冲突检测
- 时间有效性检测
- 场地容量检测
- 设备要求检测
- 已批准预约冲突
- 用户时间冲突
- 时间间隔要求

### 5. 智能推荐
- 根据时间、人数、设备需求推荐场地
- 考虑场地评分和历史使用情况
- 提供多个推荐选项

### 6. 数据导出
- 支持导出预约记录到Excel
- 自定义导出字段和时间范围
- 批量导出功能

### 7. RESTful API
- 完整的API接口
- Swagger/ReDoc自动生成文档
- 支持API认证

## 项目亮点

1. **智能冲突检测**：自动检测多种类型的冲突，确保预约合理性

## 📞 常见问题

### Q1: git push时提示"fatal: unable to access"

**解决方案**：
- 检查网络连接
- 使用Gitee代替GitHub（国内访问更快）
- 配置代理：`git config --global http.proxy http://代理地址:端口`

### Q2: git push时提示"Permission denied"

**解决方案**：
- GitHub：使用Personal Access Token代替密码
- 检查是否有仓库写入权限

### Q3: 视频文件太大，无法上传到GitHub

**解决方案**：
- GitHub单文件限制100MB
- 使用GitHub Release上传（限制2GB）
- 或上传到视频平台（B站、YouTube）

### Q4: 如何生成GitHub Personal Access Token

**步骤**：
1. GitHub → Settings → Developer settings → Personal access tokens
2. Generate new token (classic)
3. 勾选 `repo` 权限
4. 生成并保存token（只显示一次）
5. 使用token作为密码
