# CampusSpace 推荐算法

**版本**：v1.0  
**更新日期**：2026-07-17

---

## 算法目标

根据用户的使用需求（时间、人数、设备等），从所有场地中推荐最合适的场地，并按匹配度排序。

---

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| start_time | datetime | 是 | 开始时间 |
| end_time | datetime | 是 | 结束时间 |
| participant_count | int | 是 | 参与人数 |
| equipment_ids | list[int] | 否 | 所需设备ID列表 |
| venue_type | str | 否 | 场地类型 |
| building | str | 否 | 楼宇偏好 |
| exclude_venue_ids | list[int] | 否 | 排除的场地ID |

---

## 算法流程

```
输入用户需求
  ↓
筛选候选场地
  ↓
计算匹配度评分
  ↓
排序推荐结果
  ↓
返回推荐列表
```

## 1. 筛选候选场地

### 1.1 基础筛选

**筛选条件**：
- 场地状态为可用（status='available'）
- 容量满足人数要求（capacity >= participant_count）
- 无时间冲突（维修、课表、预约）

```python
def get_candidate_venues(start_time, end_time, participant_count, 
                         equipment_ids=None, venue_type=None, building=None,
                         exclude_venue_ids=None):
    """
    获取候选场地
    
    Returns:
        QuerySet: 候选场地列表
    """
    # 基础查询
    venues = Venue.objects.filter(
        status='available',
        capacity__gte=participant_count
    )
    
    # 场地类型筛选
    if venue_type:
        venues = venues.filter(venue_type=venue_type)
    
    # 楼宇筛选
    if building:
        venues = venues.filter(building=building)
    
    # 设备筛选
    if equipment_ids:
        for equipment_id in equipment_ids:
            venues = venues.filter(equipment__id=equipment_id)
    
    # 排除指定场地
    if exclude_venue_ids:
        venues = venues.exclude(id__in=exclude_venue_ids)
    
    # 冲突筛选
    available_venue_ids = []
    for venue in venues:
        # 检测冲突
        result = check_all_conflicts(venue, start_time, end_time)
        if not result['has_conflict']:
            available_venue_ids.append(venue.id)
    
    venues = venues.filter(id__in=available_venue_ids)
    
    return venues
```

## 2. 计算匹配度评分

### 2.1 评分维度

| 维度 | 权重 | 说明 |
|-----|------|------|
| 容量匹配度 | 35% | 场地容量与人数的匹配程度 |
| 设备匹配度 | 35% | 场地设备与需求的匹配程度 |
| 类型匹配度 | 20% | 场地类型与需求的匹配程度 |
| 历史偏好度 | 10% | 用户历史使用偏好 |

**说明**：
- 不考虑场地利用率平衡
- 考虑用户历史使用偏好（用户过去常使用的场地优先级稍高）
- 设备故障时，降低设备匹配度评分

### 2.2 容量匹配度

**计算公式**：
```
容量匹配度 = 1 - (场地容量 - 需求人数) / 场地容量
```

**说明**：
- 容量刚好匹配：评分 = 1.0
- 容量稍大：评分较高（如：容量100，需求80 → 0.8）
- 容量过大：评分较低（如：容量200，需求50 → 0.75）

```python
def calculate_capacity_score(venue, participant_count):
    """
    计算容量匹配度
    
    Returns:
        float: 0.0 - 1.0
    """
    if venue.capacity < participant_count:
        return 0.0
    
    # 容量刚好匹配
    if venue.capacity == participant_count:
        return 1.0
    
    # 容量匹配度
    score = 1 - (venue.capacity - participant_count) / venue.capacity
    
    return score
```

### 2.3 设备匹配度

**计算公式**：
```
设备匹配度 = 匹配设备数 / 需求设备数
```

```python
def calculate_equipment_score(venue, required_equipment_ids):
    """
    计算设备匹配度
    
    Returns:
        float: 0.0 - 1.0
    """
    if not required_equipment_ids:
        return 1.0  # 无设备需求，满分
    
    # 获取场地设备
    venue_equipment_ids = set(
        venue.venue_equipment_set.values_list('equipment_id', flat=True)
    )
    required_ids = set(required_equipment_ids)
    
    # 计算匹配设备数
    matched_count = len(venue_equipment_ids & required_ids)
    required_count = len(required_ids)
    
    return matched_count / required_count
```

### 2.4 类型匹配度

**计算公式**：
```
类型匹配度 = 1.0（匹配）或 0.7（不匹配但可用）
```

```python
def calculate_type_score(venue, preferred_type):
    """
    计算类型匹配度
    
    Returns:
        float: 0.7 - 1.0
    """
    if not preferred_type:
        return 1.0
    
    if venue.venue_type == preferred_type:
        return 1.0
    else:
        return 0.7  # 类型不匹配但可用
```

### 2.5 历史偏好度

**计算公式**：
```
历史偏好度 = 用户使用该场地的次数 / 用户总预约次数
```

**说明**：考虑用户历史使用偏好，用户过去常使用的场地优先级稍高

```python
def calculate_history_preference_score(venue, user, days=90):
    """
    计算历史偏好度
    
    Args:
        venue: 场地对象
        user: 用户对象
        days: 统计天数
    
    Returns:
        float: 0.0 - 1.0
    """
    now = timezone.now()
    start_date = now - timedelta(days=days)
    
    # 查询用户总预约次数
    total_bookings = Booking.objects.filter(
        user=user,
        start_time__range=[start_date, now]
    ).count()
    
    if total_bookings == 0:
        return 0.5  # 无历史记录，返回中等分数
    
    # 查询用户使用该场地的次数
    venue_bookings = Booking.objects.filter(
        user=user,
        venue=venue,
        start_time__range=[start_date, now]
    ).count()
    
    # 偏好度
    score = venue_bookings / total_bookings
    
    return score
```

### 2.6 设备状态影响

**说明**：设备故障时，降低设备匹配度评分

```python
def adjust_equipment_score_for_fault(equipment_score, venue):
    """
    根据设备状态调整设备匹配度评分
    
    Args:
        equipment_score: 原始设备匹配度评分
        venue: 场地对象
    
    Returns:
        float: 调整后的评分
    """
    # 检查是否有故障设备
    faulty_equipment = venue.venue_equipment_set.filter(
        status__in=['broken', 'maintenance']
    ).exists()
    
    if faulty_equipment:
        # 有故障设备，降低评分
        return equipment_score * 0.5
    
    return equipment_score
```

## 3. 生成推荐理由

```python
def generate_recommendation_reason(venue, scores, participant_count, equipment_ids):
    """
    生成推荐理由
    
    Returns:
        list: 推荐理由列表
    """
    reasons = []
    
    # 容量匹配理由
    if scores['capacity_score'] == 1.0:
        reasons.append(f'容量刚好满足{participant_count}人需求')
    elif scores['capacity_score'] >= 0.8:
        reasons.append(f'容量{venue.capacity}人，满足{participant_count}人需求')
    
    # 设备匹配理由
    if scores['equipment_score'] == 1.0:
        reasons.append('设备齐全，完全满足需求')
    elif scores['equipment_score'] >= 0.8:
        reasons.append('设备基本满足需求')
    elif scores['equipment_score'] > 0:
        # 列出缺少的设备
        venue_equipment_ids = set(
            venue.venue_equipment_set.values_list('equipment_id', flat=True)
        )
        missing_ids = set(equipment_ids) - venue_equipment_ids
        missing_names = Equipment.objects.filter(id__in=missing_ids).values_list('name', flat=True)
        reasons.append(f'缺少设备：{", ".join(missing_names)}')
    
    # 利用率理由
    if scores['utilization_score'] >= 0.7:
        reasons.append('场地利用率较低，推荐使用')
    
    return reasons
```

## 4. 主推荐函数

```python
def recommend_venues(start_time, end_time, participant_count, 
                     equipment_ids=None, venue_type=None, building=None,
                     limit=10, weights=None):
    """
    推荐场地
    
    Args:
        start_time: 开始时间
        end_time: 结束时间
        participant_count: 参与人数
        equipment_ids: 所需设备ID列表
        venue_type: 场地类型
        building: 楼宇偏好
        limit: 返回数量限制
        weights: 权重配置
    
    Returns:
        list: 推荐场地列表
        [
            {
                'venue': Venue对象,
                'score': 0.95,
                'reasons': ['容量刚好满足', '设备齐全'],
                'scores': {
                    'total_score': 0.95,
                    'capacity_score': 1.0,
                    'equipment_score': 1.0,
                    ...
                }
            },
            ...
        ]
    """
    # 1. 获取候选场地
    candidates = get_candidate_venues(
        start_time, end_time, participant_count,
        equipment_ids, venue_type, building
    )
    
    # 2. 计算评分
    recommendations = []
    for venue in candidates:
        scores = calculate_total_score(
            venue, participant_count, equipment_ids, venue_type, weights
        )
        
        reasons = generate_recommendation_reason(
            venue, scores, participant_count, equipment_ids
        )
        
        recommendations.append({
            'venue': venue,
            'score': scores['total_score'],
            'reasons': reasons,
            'scores': scores,
        })
    
    # 3. 按评分排序
    recommendations.sort(key=lambda x: x['score'], reverse=True)
    
    # 4. 限制返回数量
    recommendations = recommendations[:limit]
    
    return recommendations
```

## 5. 模糊匹配策略

当没有完全匹配的场地时，提供相近推荐：

```python
def recommend_with_fuzzy_match(start_time, end_time, participant_count, **kwargs):
    """
    带模糊匹配的推荐
    
    Returns:
        {
            'exact_matches': [],  # 完全匹配
            'fuzzy_matches': [],  # 模糊匹配
            'message': '推荐说明'
        }
    """
    # 1. 尝试完全匹配
    exact_matches = recommend_venues(
        start_time, end_time, participant_count,
        equipment_ids=kwargs.get('equipment_ids'),
        venue_type=kwargs.get('venue_type'),
        limit=5
    )
    
    # 过滤出完全匹配的（设备匹配度=1）
    exact_matches = [
        m for m in exact_matches 
        if m['scores']['equipment_score'] == 1.0
    ]
    
    if exact_matches:
        return {
            'exact_matches': exact_matches,
            'fuzzy_matches': [],
            'message': f'找到{len(exact_matches)}个完全匹配的场地'
        }
    
    # 2. 模糊匹配：放宽设备要求
    fuzzy_matches = recommend_venues(
        start_time, end_time, participant_count,
        equipment_ids=None,  # 不限制设备
        venue_type=kwargs.get('venue_type'),
        limit=5
    )
    
    # 标记缺少的设备
    for match in fuzzy_matches:
        if match['scores']['equipment_score'] < 1.0:
            match['reasons'].append('⚠️ 部分设备缺失，请自备或调整需求')
    
    return {
        'exact_matches': [],
        'fuzzy_matches': fuzzy_matches,
        'message': '未找到完全匹配的场地，以下为相近推荐'
    }
```

```python
from django.core.cache import cache

def recommend_venues_with_cache(start_time, end_time, participant_count, **kwargs):
    """
    带缓存的推荐
    """
    # 生成缓存键
    cache_key = f"recommend:{start_time}:{end_time}:{participant_count}:{kwargs}"
    
    # 尝试从缓存获取
    result = cache.get(cache_key)
    if result:
        return result
    
    # 计算推荐
    result = recommend_venues(start_time, end_time, participant_count, **kwargs)
    
    # 缓存结果（5分钟）
    cache.set(cache_key, result, 300)
    
    return result
```

```python
@api_view(['POST'])
def recommend_venues_api(request):

    data = [

            'venue_name': r['venue'].name,
            'capacity': r['venue'].capacity,
            'score': r['score'],
            'reasons': r['reasons'],
        }
        for r in recommendations
    ]
    
    return Response({
        'success': True,
        'data': data,
    })
```
