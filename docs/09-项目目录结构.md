## 命名规范

### 文件命名

- Python文件：小写+下划线，如 `views.py`, `models.py`
- 模板文件：小写+下划线，如 `venue_list.html`
- 静态文件：小写+连字符，如 `venue-list.css`

### 类命名

- 模型类：大驼峰，如 `Venue`, `Booking`
- 视图类：大驼峰+后缀，如 `VenueListView`
- 表单类：大驼峰+后缀，如 `BookingForm`

### 函数命名

- 视图函数：小写+下划线，如 `venue_list()`
- 工具函数：小写+下划线，如 `check_conflict()`

### 变量命名

- 普通变量：小写+下划线，如 `venue_list`
- 常量：大写+下划线，如 `MAX_CAPACITY`

## Git分支策略

```
main (生产分支)
  └── develop (开发分支)
        ├── feature/user-auth (功能分支)
        ├── feature/venue-management
        ├── feature/booking-system
        ├── feature/recommendation
        └── bugfix/conflict-detection (修复分支)
```

## 开发工作流

1. 从develop分支创建feature分支
2. 在feature分支开发功能
3. 编写单元测试
4. 提交代码并通过代码审查
5. 合并到develop分支
6. 测试通过后合并到main分支
7. 部署到生产环境
