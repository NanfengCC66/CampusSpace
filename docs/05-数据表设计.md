## 6. 预约表 (bookings_booking)

| 字段名 | 字段类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| booking_no | VARCHAR(50) | UNIQUE, NOT NULL | 预约编号 |
| venue_id | INT | FK, NOT NULL | 场地ID |
| user_id | INT | FK, NOT NULL | 预约人ID |
| title | VARCHAR(200) | NOT NULL | 使用目的/活动名称 |
| booking_type | VARCHAR(20) | NOT NULL | 预约用途：teaching/meeting/activity/self_study |
| start_time | DATETIME | NOT NULL | 开始时间 |
| end_time | DATETIME | NOT NULL | 结束时间 |
| participant_count | INT | NOT NULL | 参与人数 |
| contact_name | VARCHAR(100) | | 联系人 |
| contact_phone | VARCHAR(20) | | 联系电话 |
| remark | TEXT | | 备注 |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态：pending/approved/rejected/cancelled |
| priority | INT | DEFAULT 0 | 优先级（教师>学生） |
| cancel_reason | TEXT | | 取消原因 |
| cancelled_by_id | INT | FK | 取消人ID |
| cancelled_at | DATETIME | | 取消时间 |
| is_modified | BOOLEAN | DEFAULT FALSE | 是否被修改过 |
| original_booking_id | INT | FK | 原始预约ID（修改前的版本） |
| created_at | DATETIME | AUTO | 创建时间 |
| updated_at | DATETIME | AUTO | 更新时间 |

**索引**：
- idx_booking_no: booking_no
- idx_venue: venue_id
- idx_user: user_id
- idx_status: status
- idx_time: start_time, end_time

**外键**：
- venue_id → venues_venue.id
- user_id → users_user.id
- cancelled_by_id → users_user.id

**业务规则**：
- booking_no格式：BK + YYYYMMDD + 6位序号（如：BK20260717000001）
- end_time > start_time
- start_time > created_at（预约时间必须在未来）
- 预约时长：最短30分钟，最长8小时
- 同一用户每天最多预约3次或总时长不超过12小时
- 预约用途：teaching(教学)/meeting(会议)/activity(活动)/self_study(自习)
- 优先级：教师预约priority=1，学生预约priority=0
- 取消时限：开始前2小时内不能取消

## 7. 固定课表表 (schedules_schedule)

| 字段名 | 字段类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| venue_id | INT | FK, NOT NULL | 场地ID |

| day_of_week | INT | NOT NULL | 周几（1-7，周一到周日） |
| period | VARCHAR(20) | NOT NULL | 节次（如：1-2, 3-4, 5-6等） |
| start_week | INT | NOT NULL | 开始周次 |
| end_week | INT | NOT NULL | 结束周次 |
| class_name | VARCHAR(100) | | 班级 |
| student_count | INT | | 学生人数 |
| semester | VARCHAR(50) | NOT NULL | 学期（如：2025-2026-2） |
| remark | TEXT | | 备注 |

| updated_at | DATETIME | AUTO | 更新时间 |
| created_by_id | INT | FK | 创建人 |

- idx_venue: venue_id

**外键**：
- venue_id → venues_venue.id

**业务规则**：
- end_week >= start_week

## 8. 维修停用表 (maintenance_maintenance)

| 字段名 | 字段类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| venue_id | INT | FK, NOT NULL | 场地ID |
| reason | VARCHAR(200) | NOT NULL | 停用原因 |
| start_time | DATETIME | NOT NULL | 开始时间 |
| end_time | DATETIME | NOT NULL | 预计结束时间 |
| actual_end_time | DATETIME | | 实际结束时间 |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态：pending/approved/ongoing/completed/rejected |
| approval_status | VARCHAR(20) | DEFAULT 'pending' | 审批状态：pending/approved/rejected |
| approved_by_id | INT | FK | 审批人ID |
| approved_at | DATETIME | | 审批时间 |
| remark | TEXT | | 备注 |
| created_at | DATETIME | AUTO | 创建时间 |
| updated_at | DATETIME | AUTO | 更新时间 |
| created_by_id | INT | FK | 创建人（维修人员） |

**索引**：
- idx_venue: venue_id
- idx_status: status
- idx_time: start_time, end_time

**外键**：
- venue_id → venues_venue.id
- created_by_id → users_user.id
- approved_by_id → users_user.id

**业务规则**：
- end_time > start_time
- actual_end_time >= start_time（实际结束时间）
- 维修申请需要审批
- 维修期间已有预约自动取消并通知用户（提前24小时通知）
- 维修完成后，维修人员提交恢复申请

| 字段名 | 字段类型 | 约束 | 说明 |
|--------|---------|------|------|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| booking_id | INT | FK, NOT NULL | 预约ID |

| action | VARCHAR(20) | NOT NULL | 操作：approve/reject |

- booking_id → bookings_booking.id

## 数据关系图

```
users_user (用户)
  ├─ 1:N → bookings_booking (预约)
  ├─ 1:N → approvals_approval (审批记录)
  ├─ 1:N → logs_operationlog (操作日志)
  └─ 1:N → notifications_notification (通知)

venues_venue (场地)
  ├─ M:N → venues_equipment (设备)
  ├─ 1:N → venues_venueimage (场地图片)
  ├─ 1:N → bookings_booking (预约)
  ├─ 1:N → schedules_schedule (固定课表)
  └─ 1:N → maintenance_maintenance (维修停用)

bookings_booking (预约)
  └─ 1:1 → approvals_approval (审批记录)
```

## 数据库配置

### Django settings.py 配置

```python
# 开发环境 - SQLite
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}

# 生产环境 - MySQL
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': 'campusspace',
        'USER': 'your_username',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '3306',
        'OPTIONS': {
            'charset': 'utf8mb4',
        }
    }
}

# 生产环境 - PostgreSQL
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql',
        'NAME': 'campusspace',
        'USER': 'your_username',
        'PASSWORD': 'your_password',
        'HOST': 'localhost',
        'PORT': '5432',
    }
```

## 数据迁移策略

### 初始化数据

1. 创建超级管理员账号
2. 导入设备类型预置数据
3. 导入场地基础信息
4. 导入固定课表数据（如有）

### 数据备份

- 定期备份数据库（每日自动备份）
- 重要操作前手动备份
- 保留最近7天的备份文件

### 数据迁移

- 使用Django migrations管理数据库变更
- 生产环境迁移前先在测试环境验证
- 大数据量迁移使用批量操作优化性能
