## 状态定义

| 状态 | 英文 | 说明 | 可见性 |
|-----|------|------|--------|
| 待审批 | pending | 用户提交预约后的初始状态 | 用户、管理员可见 |
| 已通过 | approved | 管理员审批通过 | 用户、管理员可见 |
| 已拒绝 | rejected | 管理员审批拒绝 | 用户、管理员可见 |
| 已取消 | cancelled | 用户或管理员取消 | 用户、管理员可见 |
| 已完成 | completed | 预约时间已过，使用完成 | 历史记录可见 |
| 已过期 | expired | 待审批状态超时未审批 | 历史记录可见 |

## 状态转换规则

### 1. 创建预约

**触发条件**：用户提交预约申请  
**前置状态**：无（新建）  
**目标状态**：待审批（pending）  
**执行操作**：
- 创建预约记录
- 状态设置为pending
- 发送通知给管理员
- 记录操作日志

**代码示例**：
```python
def create_booking(user, venue, start_time, end_time, **kwargs):
    # 冲突检测
    if has_conflict(venue, start_time, end_time):
        raise ConflictError("时间冲突")
    
    # 创建预约
    booking = Booking.objects.create(
        venue=venue,
        user=user,
        start_time=start_time,
        end_time=end_time,
        status='pending',
        **kwargs
    )
    
    # 发送通知
    send_notification_to_admins(booking)
    
    return booking
```

---

### 2. 审批通过

**触发条件**：管理员点击"通过"  
**前置状态**：待审批（pending）  
**目标状态**：已通过（approved）  
**执行操作**：
- 再次检测冲突（防止并发）
- 更新状态为approved
- 创建审批记录
- 发送通知给用户
- 记录操作日志

**代码示例**：
```python
def approve_booking(booking, approver, comment=''):
    # 检查状态
    if booking.status != 'pending':
        raise StatusError("只能审批待审批状态的预约")
    
    # 再次检测冲突
    if has_conflict(booking.venue, booking.start_time, booking.end_time):
        raise ConflictError("时间冲突，请重新审批")
    
    # 更新状态
    booking.status = 'approved'
    booking.save()
    
    # 创建审批记录
    Approval.objects.create(
        booking=booking,
        approver=approver,
        action='approve',
        comment=comment
    )
    
    # 发送通知
    send_notification(booking.user, "您的预约已通过审批")
```

### 3. 审批拒绝

**触发条件**：管理员点击"拒绝"  
**前置状态**：待审批（pending）  
**目标状态**：已拒绝（rejected）  
**执行操作**：
- 更新状态为rejected
- 创建审批记录（包含拒绝理由）
- 发送通知给用户
- 记录操作日志

**代码示例**：
```python
def reject_booking(booking, approver, reason):
    # 检查状态
    if booking.status != 'pending':

    booking.status = 'rejected'
    booking.save()
    
    # 创建审批记录

        booking=booking,

        action='reject',
        comment=reason
    )
    
    # 发送通知
    send_notification(booking.user, f"您的预约被拒绝：{reason}")
```

### 4. 用户取消预约

**触发条件**：用户点击"取消预约"  
**前置状态**：待审批（pending）或 已通过（approved）  
**目标状态**：已取消（cancelled）  
**执行操作**：
- 检查是否可取消（未开始、符合时限）
- 更新状态为cancelled
- 记录取消原因和取消人
- 发送通知给管理员（如已通过）
- 记录操作日志

**代码示例**：
```python
def cancel_booking_by_user(booking, user, reason=''):
    # 检查权限
    if booking.user != user:
        raise PermissionError("只能取消自己的预约")
    
    # 检查状态
    if booking.status not in ['pending', 'approved']:
        raise StatusError("当前状态不可取消")
    
    # 检查时间
    if booking.start_time <= timezone.now():
        raise TimeError("已开始的预约不可取消")
    
    # 更新状态
    booking.status = 'cancelled'
    booking.cancel_reason = reason
    booking.cancelled_by = user
    booking.cancelled_at = timezone.now()
    booking.save()
    
    # 发送通知
    if booking.status == 'approved':
        send_notification_to_admins(f"预约{booking.booking_no}已被用户取消")
```

### 6. 预约完成

**触发条件**：预约结束时间到达（定时任务）  
**前置状态**：已通过（approved）  
**目标状态**：已完成（completed）  
**执行操作**：
- 更新状态为completed
- 记录完成时间
- 更新统计数据

**代码示例**：
```python
def complete_booking(booking):
    # 检查状态
    if booking.status != 'approved':
        return
    
    # 检查时间
    if booking.end_time > timezone.now():
        return
    
    # 更新状态
    booking.status = 'completed'
    booking.save()
```

---

### 7. 预约过期

**触发条件**：待审批超时（定时任务）  
**前置状态**：待审批（pending）  
**目标状态**：已过期（expired）  
**执行操作**：
- 检查是否超时（如：24小时未审批）
- 更新状态为expired
- 发送通知给用户和管理员

**代码示例**：
```python
def expire_booking(booking, timeout_hours=24):
    # 检查状态
    if booking.status != 'pending':
        return
    
    # 检查是否超时
    timeout = booking.created_at + timedelta(hours=timeout_hours)
    if timezone.now() < timeout:
        return
    
    # 更新状态
    booking.status = 'expired'
    booking.save()
    
    # 发送通知
    send_notification(booking.user, "您的预约已超时未审批，请重新提交")
    send_notification_to_admins(f"预约{booking.booking_no}已超时未审批")
```

## 状态转换矩阵

| 当前状态 | 用户取消 | 管理员取消 | 审批通过 | 审批拒绝 | 时间到达 | 超时 |
|---------|---------|-----------|---------|---------|---------|------|
| pending | ✓ → cancelled | ✓ → cancelled | ✓ → approved | ✓ → rejected | ✗ | ✓ → expired |
| approved | ✓ → cancelled | ✓ → cancelled | ✗ | ✗ | ✓ → completed | ✗ |
| rejected | ✗ | ✓ → cancelled | ✗ | ✗ | ✗ | ✗ |
| cancelled | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ |
| completed | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ |
| expired | ✗ | ✓ → cancelled | ✗ | ✗ | ✗ | ✗ |

**图例**：
- ✓：允许转换
- ✗：不允许转换
- →：目标状态

## 定时任务

### 1. 更新已完成状态

**执行频率**：每5分钟  
**任务内容**：检查已通过的预约，结束时间已过的更新为已完成

```python
@task
def update_completed_bookings():
    now = timezone.now()
    bookings = Booking.objects.filter(
        status='approved',
        end_time__lte=now
    )
    for booking in bookings:
        booking.status = 'completed'
        booking.save()
```

### 2. 检查过期预约

**执行频率**：每小时  
**任务内容**：检查待审批的预约，超时的更新为已过期

```python
@task
def check_expired_bookings():
    timeout_hours = getattr(settings, 'APPROVAL_TIMEOUT_HOURS', 24)
    timeout = timezone.now() - timedelta(hours=timeout_hours)
    
    bookings = Booking.objects.filter(
        status='pending',
        created_at__lte=timeout
    )
    for booking in bookings:
        booking.status = 'expired'
        booking.save()
        # 发送通知
        send_notification(booking.user, "预约已超时")
```

### 3. 预约提醒

**执行频率**：每小时  
**任务内容**：检查即将开始的预约，发送提醒通知

```python
@task
def send_booking_reminders():
    # 提前1小时提醒
    remind_time = timezone.now() + timedelta(hours=1)
    
    bookings = Booking.objects.filter(
        status='approved',
        start_time__range=[timezone.now(), remind_time]
    )
    for booking in bookings:
        send_notification(booking.user, f"您的预约即将开始：{booking}")
```

## 异常处理

### 并发冲突处理

**场景**：多个管理员同时审批同一预约  
**解决方案**：使用数据库事务和乐观锁

```python
from django.db import transaction

@transaction.atomic
def approve_booking_safe(booking_id, approver):
    # 使用select_for_update锁定记录
    booking = Booking.objects.select_for_update().get(id=booking_id)
    
    if booking.status != 'pending':
        raise StatusError("预约状态已变更")
    
    # 审批操作
    booking.status = 'approved'
    booking.save()
```

### 状态不一致处理

**场景**：系统异常导致状态不一致  
**解决方案**：定期检查和修复

```python
@task
def fix_inconsistent_status():
    # 检查已通过但时间已过的预约
    Booking.objects.filter(
        status='approved',
        end_time__lt=timezone.now()
    ).update(status='completed')
    
    # 检查待审批但超时的预约
    timeout = timezone.now() - timedelta(hours=24)
    Booking.objects.filter(
        status='pending',
        created_at__lt=timeout
    ).update(status='expired')
```
