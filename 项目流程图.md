## 三、用户操作流程图

### 3.1 预约流程

```mermaid
sequenceDiagram
    participant 用户
    participant 系统
    participant 推荐引擎
    participant 冲突检测
    participant 管理员
    
    用户->>系统: 登录系统
    系统->>用户: 显示首页
    
    用户->>系统: 选择预约功能
    系统->>用户: 显示预约表单
    
    用户->>推荐引擎: 请求场地推荐
    推荐引擎->>系统: 返回推荐场地列表
    系统->>用户: 显示推荐结果
    
    用户->>系统: 提交预约申请
    系统->>冲突检测: 检查时间冲突
    
    alt 无冲突
        冲突检测->>系统: 冲突检测通过
        系统->>系统: 创建预约记录
        系统->>管理员: 发送审批通知
        系统->>用户: 显示预约成功
    else 有冲突
        冲突检测->>系统: 检测到冲突
        系统->>用户: 显示冲突提示
    end
    
    管理员->>系统: 审批预约
    系统->>用户: 发送审批结果通知
```

### 3.2 审批流程

```mermaid
sequenceDiagram
    participant 管理员
    participant 系统
    participant 冲突检测
    participant 用户
    participant 通知服务
    
    管理员->>系统: 查看待审批列表
    系统->>管理员: 显示待审批预约
    
    管理员->>系统: 选择预约进行审批
    系统->>冲突检测: 再次检查冲突
    
    alt 通过审批
        冲突检测->>系统: 无冲突
        管理员->>系统: 点击通过
        系统->>系统: 更新预约状态为已通过
        系统->>通知服务: 发送通知
        通知服务->>用户: 预约已通过
    else 拒绝审批
        管理员->>系统: 点击拒绝
        系统->>系统: 更新预约状态为已拒绝
        系统->>通知服务: 发送通知
        通知服务->>用户: 预约被拒绝
    end
```

## 七、智能推荐算法流程图

```mermaid
graph TB
    Start[开始推荐] --> Input[输入预约需求]
    Input --> Filter1[筛选可用场地]
    
    Filter1 --> Check1{检查容量}
    Check1 --> |容量不足| Exclude1[排除]
    Check1 --> |容量足够| Filter2[筛选时间段可用场地]
    
    Filter2 --> Check2{检查时间可用性}
    Check2 --> |时间冲突| Exclude2[排除]
    Check2 --> |时间可用| Score[计算评分]
    
    Score --> Score1[容量匹配评分<br/>35%]
    Score --> Score2[设备匹配评分<br/>25%]
    Score --> Score3[楼宇偏好评分<br/>15%]
    Score --> Score4[连续性评分<br/>10%]
    Score --> Score5[节能评分<br/>10%]
    
    Score1 --> Total[总分计算]
    Score2 --> Total
    Score3 --> Total
    Score4 --> Total
    Score5 --> Total
    
    Total --> Sort[按分数排序]
    Sort --> Top[取前3个结果]
    Top --> End[返回推荐结果]
```

## 八、冲突检测流程图

```mermaid
graph TB
    Start[开始检测] --> Input[输入预约信息]
    Input --> Check1[检查预约冲突]
    
    Check1 --> Query1[查询已通过预约]
    Query1 --> Compare1[比较时间段]
    Compare1 --> Result1{是否有冲突}
    
    Result1 --> |有冲突| Conflict1[记录预约冲突]
    Result1 --> |无冲突| Check2[检查待审批预约]
    
    Check2 --> Query2[查询待审批预约]
    Query2 --> Compare2[比较时间段]
    Compare2 --> Result2{是否有冲突}
    
    Result2 --> |有冲突| Conflict2[记录预约冲突]
    Result2 --> |无冲突| Check3[检查固定课表]
    
    Check3 --> Query3[查询课表安排]
    Query3 --> Compare3[比较时间段]
    Compare3 --> Result3{是否有冲突}
    
    Result3 --> |有冲突| Conflict3[记录课表冲突]
    Result3 --> |无冲突| Check4[检查维修记录]
    
    Check4 --> Query4[查询维修记录]
    Query4 --> Compare4[比较时间段]
    Compare4 --> Result4{是否有冲突}
    
    Result4 --> |有冲突| Conflict4[记录维修冲突]
    Result4 --> |无冲突| Summary[汇总冲突结果]
    
    Conflict1 --> Summary
    Conflict2 --> Summary
    Conflict3 --> Summary
    Conflict4 --> Summary
    
    Summary --> Output{是否有冲突}
    Output --> |有冲突| Return1[返回冲突列表]
    Output --> |无冲突| Return2[返回通过]
    
    Return1 --> End[结束]
    Return2 --> End
```

## 九、批量审批流程图

```mermaid
graph TB
    Start[开始批量审批] --> Select[选择多个预约]
    Select --> Validate[验证预约状态]
    
    Validate --> Loop{遍历每个预约}
    Loop --> |下一个| Check[检查冲突]
    
    Check --> Result{是否有冲突}
    Result --> |有冲突| Record1[记录失败]
    Result --> |无冲突| Approve[审批通过]
    
    Approve --> Update1[更新预约状态]
    Update1 --> Create1[创建审批记录]
    Create1 --> Record2[记录成功]
    
    Record1 --> Loop
    Record2 --> Loop
    
    Loop --> |完成| Summary[汇总结果]
    Summary --> Output[返回统计信息]
    Output --> End[结束]
```

1. **Mermaid格式**：上述流程图使用Mermaid语法编写，可在支持Mermaid的Markdown编辑器中直接渲染

2. **在线渲染**：

3. **IDE支持**：
   - VS Code：安装"Markdown Preview Mermaid Support"插件
   - JetBrains IDE：原生支持Mermaid语法

4. **文档插入**：
   - 将渲染后的图片保存
