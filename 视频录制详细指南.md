# CampusSpace 系统演示视频录制详细指南

**视频时长**：8-10分钟  
**录制工具**：OBS Studio（推荐）/ Bandicam  
**视频分辨率**：1920x1080  
**帧率**：30fps

---

## 一、录制前准备工作

### 1.1 系统准备

**步骤1：启动系统**

```bash
# 激活虚拟环境
CampusSpace\Scripts\activate

# 启动Django服务器
python manage.py runserver

# 确认系统运行在 http://127.0.0.1:8000/
```

**步骤2：准备浏览器**
- 使用Chrome或Edge浏览器
- 设置浏览器窗口大小为1920x1080
- 清除浏览器缓存（Ctrl+Shift+Delete）
- 关闭不必要的浏览器插件

**步骤3：准备演示数据**
- 确认已有数据：159用户、113场地、504预约
- 演示账号：
  - 管理员：admin / admin123
  - 教师：teacher / teacher123
  - 学生：student / student123

### 1.2 录屏工具配置（OBS Studio）

**步骤1：安装OBS Studio**
- 下载地址：https://obsproject.com/
- 安装完成后打开OBS

**步骤2：配置录制参数**

```
设置 → 输出：
- 录制格式：mp4
- 编码器：x264
- 码率：6000 Kbps

设置 → 视频：
- 输出分辨率：1920x1080
- 帧率：30 FPS

设置 → 音频：
- 采样率：44.1 kHz
- 声道：立体声
```

**步骤3：添加显示器捕获**

```
来源 → 点击"+" → 显示器捕获
- 选择主显示器
- 勾选"捕获鼠标"
- 鼠标高亮效果：启用
```

**步骤4：添加麦克风（可选）**
```
来源 → 点击"+" → 音频输入捕获
- 选择麦克风设备
- 用于实时解说配音
```

### 1.3 录制环境准备

**步骤1：关闭干扰**
- 关闭其他应用程序
- 关闭系统通知（Windows设置 → 系统 → 通知）
- 关闭杀毒软件弹窗

**步骤2：准备桌面**
- 清理桌面图标
- 设置简洁的桌面背景
- 只保留浏览器窗口

**步骤3：浏览器准备**
- 打开 http://127.0.0.1:8000/
- 按F11进入全屏模式（可选）
- 准备好登录页面

## 三、录制技巧与注意事项

### 3.1 录制技巧

**1. 分段录制**
- 按场景分段录制，便于后期剪辑
- 每个场景录制完成后停止，检查效果
- 如果不满意，重新录制该场景

**2. 操作节奏**
- 操作速度适中，不要太快
- 每个操作停留1-2秒，让观众看清楚
- 重要信息可以多停留几秒

**3. 鼠标操作**
- 移动鼠标平稳，不要抖动
- 点击操作明确，有明显的点击效果
- 使用鼠标高亮效果，便于观众跟随

**4. 等待加载**
- 页面加载时等待完全加载
- 不要在加载过程中操作
- 加载完成后停留1秒再操作

**5. 输入操作**
- 输入速度适中，不要过快
- 可以使用复制粘贴，确保输入准确
- 输入完成后停留1秒

### 3.2 注意事项

**1. 避免的操作**
- ❌ 不要展示密码输入过程（用剪映后期打码）
- ❌ 不要展示真实的个人隐私信息
- ❌ 不要在录制过程中接电话或聊天
- ❌ 不要在录制过程中移动窗口

**2. 推荐的操作**
- ✅ 提前准备好所有演示数据
- ✅ 提前登录好所有需要的账号
- ✅ 提前打开所有需要的页面
- ✅ 录制前测试一遍流程

**3. 录制环境**
- ✅ 安静的环境，无背景噪音
- ✅ 稳定的网络连接
- ✅ 足够的磁盘空间（至少10GB）
- ✅ 关闭系统通知和其他干扰

## 八、常见问题解决

### 问题1：录制时画面卡顿

**原因**：电脑性能不足或OBS设置过高

**解决**：
```
1. 降低OBS录制分辨率到1280x720
2. 降低帧率到24fps
3. 关闭其他应用程序
4. 使用硬件编码器（NVENC）代替软件编码器
```

### 问题2：录制时没有声音

**原因**：麦克风未正确配置

**解决**：
```
1. 检查OBS音频设置
2. 确认麦克风设备已选择
3. 检查系统音量设置
4. 测试麦克风是否正常工作
```

### 问题3：导出视频文件过大

**原因**：码率设置过高

**解决**：
```
1. 降低码率到3000-5000 Kbps
2. 使用H.264编码
3. 降低分辨率到1280x720（如需要）
```

### 问题4：页面加载太慢

**原因**：系统性能或网络问题

**解决**：
```
1. 提前打开所有页面，让浏览器缓存
2. 使用本地数据，避免网络请求
3. 剪辑时删除加载等待时间
```
